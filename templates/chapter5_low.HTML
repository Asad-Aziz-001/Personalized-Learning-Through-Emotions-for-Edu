Linear Inequalities Basics

Understanding Linear Inequalities

Linear inequalities are similar to linear equations, but instead of an equals sign, they use inequality symbols like <, >, ‚â§, or ‚â•.

Linear Inequality: \[ ax + by < c, \quad ax + by > c, \quad ax + by \leq c, \quad \text{or} \quad ax + by \geq c \]

Understanding Linear Inequalities

What is a Linear Inequality?

A linear inequality is a mathematical statement that relates a linear expression as either less than, greater than, less than or equal to, or greater than or equal to another expression.

Real-World Applications

‚Ä¢ Budget constraints: If you have $100 for shopping, your spending can be represented as x ‚â§ 100

‚Ä¢ Time constraints: If you need to study at least 2 hours, your study time t can be represented as t ‚â• 2

‚Ä¢ Recipe requirements: If a recipe requires at least 3 cups of flour but no more than 5, we can write 3 ‚â§ f ‚â§ 5

Visual Representation

Unlike equations that represent a single line, inequalities represent entire regions of the coordinate plane. The boundary line divides the plane into two half-planes, and the inequality tells us which half-plane to include.

Solve: 5x - 3 < 2x + 9

Graphing Linear Inequalities

Graphing Techniques for Linear Inequalities

Learn the step-by-step process for graphing linear inequalities and identifying solution regions.

Graphing Steps: 1. Graph the boundary line (solid for ‚â§ or ‚â•, dashed for < or >)
2. Test a point not on the line (usually (0,0))
3. Shade the region that satisfies the inequality

Mastering Inequality Graphing

Why Graph Inequalities?

Graphing inequalities helps us visualize all possible solutions. Unlike equations that have specific solutions, inequalities have infinitely many solutions that form a region on the coordinate plane.

The Test Point Method

The easiest way to determine which side of the boundary line to shade is to test a point. If (0,0) is not on the line, it's usually the best choice because calculations are simple.

If (0,0) satisfies the inequality, shade the side containing (0,0). If not, shade the opposite side.

Visualizing Different Cases

Different types of inequalities produce different graphs:

‚Ä¢ y ‚â• mx + b: Solid line, shaded above

‚Ä¢ y ‚â§ mx + b: Solid line, shaded below

‚Ä¢ y > mx + b: Dashed line, shaded above

‚Ä¢ y < mx + b: Dashed line, shaded below

Systems of Linear Inequalities

Solving Systems of Linear Inequalities

Find the solution set (feasible region) for systems of linear inequalities and identify corner points.

Feasible Region: The set of all points that satisfy all inequalities in a system simultaneously.
Corner Points: Points where boundary lines intersect. These are important in optimization problems.

Understanding Systems of Inequalities

What is a System of Inequalities?

A system of linear inequalities consists of two or more linear inequalities with the same variables. The solution to the system is the intersection (overlap) of all individual solution sets.

Finding the Feasible Region

To find the feasible region:

1. Graph each inequality on the same coordinate plane

2. The feasible region is where all shaded areas overlap

3. Identify corner points where boundary lines intersect

Visualizing the Feasible Region

The feasible region can be bounded (a polygon) or unbounded. In optimization problems, we're often interested in bounded regions where we can find maximum or minimum values.

Corner points are important because in linear programming, optimal solutions occur at these points.

Linear Programming

Linear Programming Problems

Learn about optimizing linear functions subject to constraints with real-world applications.

Linear Programming Components: 1. Objective Function: The function to maximize or minimize
2. Constraints: Limitations or requirements
3. Decision Variables: The quantities we can control
4. Feasible Region: The set of all points satisfying all constraints
5. Optimal Solution: The point in the feasible region that optimizes the objective function

Understanding Linear Programming

What is Linear Programming?

Linear programming is a mathematical method to achieve the best outcome (maximum profit or lowest cost) in a mathematical model whose requirements are represented by linear relationships.

Key Theorem: Corner Point Method

If a linear programming problem has an optimal solution, it must occur at a corner point of the feasible region. This means we only need to evaluate the objective function at the corner points to find the optimal solution.

Real-World Applications

‚Ä¢ Factory Production: Maximize profit by deciding how many units of each product to make, given limited resources

‚Ä¢ Diet Planning: Create the cheapest diet that meets all nutritional requirements

‚Ä¢ Transportation: Minimize shipping costs while meeting demand

Example 3 ‚Äî Real-World Application: Diet Problem

How many units of each food should be fed to each animal at the smallest possible cost?
Step 1: Define variables:
Let x = units of food X
Let y = units of food Y
Step 2: Objective function (minimize cost):
C = 25x + 30y
Step 3: Constraints:
Protein: 2x + 3y ‚â• 12

Step 4: Find corner points:

(0,6): C=25(0)+30(6)=180
(4,0): C=25(4)+30(0)=100
(8,0): C=25(8)+30(0)=200
Step 7: Minimum cost is Rs. 100 with 4 units of food X and 0 units of food Y
